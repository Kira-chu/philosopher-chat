# 🔍 查看部署错误并修复

## 第一步：查看详细错误信息

1. **在 Deployments 页面，找到失败的部署记录**
   - 应该能看到一个红色的 "FAILED" 标签
   - 或者看到 "There was an error deploying from source" 的通知

2. **点击失败的部署记录**
   - 或者点击 "View logs"（查看日志）按钮

3. **在日志中查找错误信息**
   - 滚动查看完整的错误信息
   - 特别关注红色的错误消息

4. **把错误信息复制给我**
   - 特别是最后几行的错误信息
   - 或者截图发给我

---

## 常见错误和解决方法

### 错误 1: "Error creating build plan with Railpack"

**解决方法：**
1. 确认 Root Directory 是 `backend`（不是 `/backend`）
2. 确认 `backend/requirements.txt` 文件存在
3. 确认 `backend/Procfile` 文件存在

### 错误 2: "Module not found" 或 "Import error"

**解决方法：**
1. 检查 `backend/requirements.txt` 是否包含所有依赖
2. 确认 Root Directory 设置正确

### 错误 3: "Port already in use" 或端口相关错误

**解决方法：**
- Railway 会自动处理端口，不需要手动配置
- 检查 Procfile 中的命令是否正确

### 错误 4: "Environment variable not set"

**解决方法：**
1. 确认三个环境变量都已添加
2. 确认变量名称拼写正确

## 快速检查清单

在查看日志前，先确认：

- [ ] Root Directory 是 `backend`（在 Settings → Source 中）
- [ ] 三个环境变量都已添加（在 Variables 标签中）
- [ ] `backend/requirements.txt` 文件存在
- [ ] `backend/Procfile` 文件存在
- [ ] 代码已推送到 GitHub
