# 🔧 修复 Railway 部署失败问题

## 问题：Build failed - "Error creating build plan with Railpack"

这个错误通常是因为 Railway 没有正确检测到项目类型或配置。

---

## ✅ 解决方案（按顺序操作）

### 第一步：检查 Settings 中的 Root Directory

1. 在 Railway 项目页面，点击 **"Settings"**（设置）标签
2. 找到 **"Root Directory"**（根目录）
3. **确认是否设置为 `backend`**
   - 如果没有设置，点击 **"Edit"**，输入：`backend`
   - 点击 **"Save"**（保存）

### 第二步：检查文件是否存在

我已经创建了 `backend/railway.json` 配置文件，确保 Railway 能正确识别项目。

### 第三步：重新部署

1. 在项目页面，点击 **"Deployments"**（部署）标签
2. 找到失败的部署记录
3. 点击右侧的 **"..."**（三个点）菜单
4. 选择 **"Redeploy"**（重新部署）

或者：

1. 点击左侧的 **"philosopher-chat"** 服务卡片
2. 点击 **"Deploy"** 按钮

### 第四步：查看日志（如果还是失败）

1. 点击失败的部署记录
2. 点击 **"View logs"**（查看日志）
3. 把错误信息告诉我，我会帮你解决

---

## 📝 常见问题

### Q1: 还是显示 "Error creating build plan"

**解决方法：**
1. 确认 Root Directory 已设置为 `backend`
2. 确认 `backend/requirements.txt` 文件存在
3. 确认 `backend/Procfile` 文件存在
4. 重新部署

### Q2: 找不到 Settings 标签

**解决方法：**
1. 点击项目名称进入项目详情
2. 在顶部导航栏找到 **"Settings"** 标签

### Q3: 部署一直失败

**解决方法：**
1. 查看日志，找到具体错误信息
2. 告诉我错误信息，我会帮你解决
